# Aurelius Landing Page Blueprint

## Goal
Create a landing page that explains Aurelius clearly, shows the product value fast, and supports future enterprise sales.

## Positioning Statement
Aurelius is an operational HR Intelligence platform that helps teams detect risk, act on it, and measure outcomes in one workflow.

## Core Message
1. Find workforce risk early.
2. Explain why it matters.
3. Take action immediately.
4. Track whether the action worked.

## Primary Audience
1. CHRO.
2. CFO.
3. Head of People Analytics.
4. HRBP leaders.
5. Talent Acquisition leaders.

## Landing Page Structure
1. Hero section
2. Problem section
3. Solution section
4. Product modules
5. Demo proof section
6. Integrations section
7. Security and governance section
8. CTA section

## Hero Section Copy
Headline:
Operational HR Intelligence for faster decisions and measurable outcomes.

Subheadline:
Aurelius unifies workforce risk, interventions, compliance checks, and scenario planning in one lean platform.

Primary CTA:
Request demo

Secondary CTA:
See product workflow

## Problem Section
1. HR teams see reports too late.
2. Managers do not know which teams are at risk.
3. Actions are rarely tracked after they are taken.
4. Data is split across HRIS, ATS, and spreadsheets.

## Solution Section
1. Explainable attrition risk.
2. Intervention workflows with outcome tracking.
3. Policy checks before high-impact actions.
4. Workforce scenarios and budget planning.

## Product Modules
1. Risk and Retention Command Center
2. Talent Mobility and Hiring Intelligence
3. Workforce Economics and Planning
4. Trust, Safety, and Compliance

## Proof / Value Points
1. Reduce surprise attrition.
2. Improve retention action timing.
3. Track intervention impact.
4. Save hiring and vacancy cost.
5. Keep auditability and governance in place.

## Demo Section
Show:
1. Import data.
2. Train model.
3. Inspect risk drivers.
4. Create intervention.
5. Run scenario.
6. Review outcomes.

## Integrations Section
Describe support for:
1. HRIS
2. ATS
3. Engagement
4. Productivity metadata
5. Finance and planning data

Use language that makes live vendor auth optional, not required for the first launch.

## Security and Governance Section
1. Tenant-scoped access.
2. Policy checks.
3. Audit trail.
4. Confidence and evidence on recommendations.
