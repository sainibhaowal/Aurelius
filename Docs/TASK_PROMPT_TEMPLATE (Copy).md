Use this short prompt before giving any phase, feature, fix, page, backend, frontend, or integration task.

## Short Universal Prompt

Build a proper implementation plan first before coding.

Make sure you clearly identify:

1. What to do
2. Why to do it
3. Where to do it
4. Which exact files, folders, routes, APIs, components, schemas, tests, and docs are involved
5. How to implement it safely and accurately
6. What must not break in the existing system

Work in a production-safe way.

Always protect:

1. Existing code structure
2. Existing integrations and connections
3. Auth, tenant isolation, security, encryption, and cryptography-related safety
4. Existing backend and frontend behavior
5. Existing workflows, APIs, jobs, storage, memory, evolution, and events

Before implementation:

1. Check the current codebase carefully
2. Find exact paths and dependencies
3. Build a clear phase plan
4. Identify risks, gaps, and validation steps
5. Wait for approval if this task is plan-first

During implementation:

1. Do not miss important functionality
2. Do not damage other files, logic, or workflows
3. Keep behavior accurate, secure, and production-grade
4. Implement end to end, not partially

Before claiming completion:

1. Run tests and verification
2. Check regressions
3. Update documentation
4. Confirm everything is working accurately
5. Commit and tag only after verified success

## One-Line Compact Version

Plan first, identify exact files and risks, implement only in the approved scope, protect security/integrations/existing behavior, test and verify end to end, update docs, and only then claim completion and commit/tag.
